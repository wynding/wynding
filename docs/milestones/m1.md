## What M1 is

M1 is the **first vertical slice** — the whole Phase-1 core loop from
[PRD 0001](../prd/0001-core-gameplay.md), instantiated at its thinnest and playable end to end:
**one board · one tower · one creep · one wave.** It is the first thing we show people (a public
**alpha**), so it ships under the full accessibility gate (ADR 0003) and externalized strings
(ADR 0004) from day one.

PRD 0001 fixed the **shapes and invariants** and deliberately deferred every **number** to
per-milestone tuning. This spec fills that in for M1 so there is **nothing to guess during
implementation**. Every quantity here is a **first-pass value, chosen to be tuned** — the point is
that a concrete, coherent, buildable set exists, not that these are final balance. Numbers live in
the ruleset (ADR 0007), so re-tuning is a `rulesetHash` bump, never a code change.

Terminology is the glossary's ([CONTEXT.md](../CONTEXT.md)); this spec introduces **no new
canonical terms**.

## The cut line

M1 makes PRD 0001's M1 slice concrete. What is **on** is the complete loop at one instance each;
what is **off** has its shape already decided in 0001 and activates in a later milestone.

| On at M1 (built now)                                                                 | Off at M1 (shape decided in 0001; activates later)                     |
| ------------------------------------------------------------------------------------ | ---------------------------------------------------------------------- |
| One **single-path board** (one entrance, one exit, open buildable field)             | AoE / slow / stun / DoT / support / burst effects (M2+)                |
| One **single-target ground tower**, single level (no upgrades)                       | Air creeps + anti-air / domain targeting (M2)                          |
| One **ground creep** (plain hit points; no armor, no immunities)                     | Tower leveling & creep leveling (M4)                                   |
| One **wave**                                                                         | Multiple / overlapping waves (M2–M3)                                   |
| Dynamic re-pathing · the maze invariant · place-adjacent-not-on-creep                | Wave-clear + early-call **bonus payouts** (values switch on at M2)     |
| Sell at a haircut (no move; juggling possible)                                        | Easy / Hard difficulty tiers (tuned at M5)                             |
| Win = survive the wave · loss = lives to 0                                            | Interest on unspent bounty (decided at M5)                             |
|                                                                                      | Audio / sound (M1 ships silent)                                        |
| Numeric **score** + **star grade**                                                   | Best-score persistence (Phase 2) · leaderboard submission (Phase 3)    |
| Speed **pause (build-while-paused) / 1× / 2×**                                        |                                                                        |
| Full **accessibility gate** + **externalized strings** (with the difficulty waiver)  |                                                                        |

The economy runs thin: **starting bounty + per-kill only.** The wave-clear and early-call bonus
*mechanics* exist but are **tuned to zero** in M1's ruleset — one wave makes them uninteresting; they
get meaningful values at M2 when multiple waves arrive. Difficulty runs on the single **Medium**
reference curve under the recorded ADR 0003 §3 waiver (PRD 0001 §10).

## Concrete spec

### Board

- A **28 × 24 cell grid** with a **1-cell blocked border**, leaving a **26 × 22 buildable-open
  field**. (Cell = the fine creep grid unit; a tower occupies a **2 × 2** block, a creep is **1 × 1**.)
  **1 cell = 1 tile = 256 fixed-point units** — "cell" and "tile" are the same unit throughout.
- **Single path:** the **entrance** is a **2-cell-tall walkable-unbuildable opening in the left
  border column at rows 11–12** (0-indexed), and the **exit** the matching opening in the **right**
  border column at rows 11–12. Those opening cells are the **only** walkable-unbuildable cells — there
  is no interior apron. The field starts empty; the player lengthens the one entrance→exit route by
  building.
- Cell classes are 0001's three: **buildable-open** (the field), **walkable-unbuildable** (the
  entrance/exit aprons), and **blocked** (the border). Movement is 8-connected with no
  corner-cutting; the maze invariant (exit reachable from the entrance **and** from every live
  creep's cell) is enforced before every build.

### The creep

The one M1 creep is the plain **ground** unit — the baseline every future creep varies from.

- **Hit points:** 20.
- **Speed:** **26 fixed-point units/tick** (≈ 2.03 tiles/s at 256 units/tile, 20 Hz).
- **Kill bounty:** 1.
- **Domain:** ground. **No armor, no immunity flags, no role** (not a boss). A leak costs **1 life**.

### The tower

The one M1 tower is the plain **single-target ground** attacker — the baseline every future tower
composes from.

- **Cost:** 5 bounty.
- **Damage:** 10 per shot (direct; armor is 0 at M1, so a creep dies in **2 hits**).
- **Range:** **4 tiles** (1024 fixed-point units), Euclidean from the 2 × 2 footprint centre.
- **Fire cadence:** one shot every **30 ticks** (1.5 s).
- **Projectile:** impact scheduled at `fire_tick + 4` (≈ 0.2 s cosmetic travel); the sim runs no
  per-tick trajectory (PRD 0001 §4).
- **Targeting:** single-target, sticky **"first"** (smallest remaining path-distance to the exit),
  carrying a fire-time snapshot; a shot is wasted if the target dies or leaks in flight.
- **Footprint:** 2 × 2, **single level (no upgrades)**, targets **ground only**.

### Economy & starting resources

- **Currency:** bounty (per-run; no persistence).
- **Starting bounty:** 80 (buys up to 16 towers — M1 is deliberately winnable with real play).
- **Income at M1:** **per-kill bounty only** (+1 per kill). The wave-clear and early-call bonuses
  are present in the ruleset but **valued at 0**.
- **Selling:** always legal, instant. **Refund = ⌊75% × cumulative spend⌋** (a 25% haircut) —
  **3 bounty** on the 5-cost M1 tower. No move action.

### Wave & timing

- **One wave: 10 creeps**, all the M1 creep above.
- **Spawn spacing:** creeps released **≈ 1 s (20 ticks) apart** (our first-pass — 0001 leaves
  intra-wave spacing to tuning), so the wave trickles out over ~9 s.
- **First-wave countdown:** ≈ **25 s (500 ticks)** before the wave auto-launches; **callable early**
  from tick 0 (the early-call bonus is 0 at M1, so calling early is a pure tempo choice here).

### Lives, win & loss

- **Starting lives:** **10.** *(20 was the other value considered — see Open questions.)* Against a
  single 10-creep wave, 20 lives is unloseable by leaking; **10 lives makes "leak the whole wave →
  0 lives → loss" a real, testable failure**, so both outcomes occur in ordinary play (an undefended
  board loses; any working defense wins).
- **Loss:** lives reach **zero or below**.
- **Win:** all 10 creeps are resolved (killed or leaked), the wave schedule is exhausted, and **≥ 1
  life remains**.

### Scoring & star grade

- **Numeric score** (deterministic, computed from sim state so the server can re-derive it per
  ADR 0006): **`Σ kill-bounties + (lives_remaining × 25)`.** First-pass; kills and the survival
  term are the only inputs live at M1 (early-send and upgrade-value inputs come online at M2/M4).
- **Star grade** (derived from lives remaining — the casual "how'd I do," never a currency, never
  gates content). First-pass thresholds against 10 starting lives:
  - **3★** — lives ≥ 9 (lost ≤ 1)
  - **2★** — lives ≥ 6 (lost ≤ 4)
  - **1★** — lives ≥ 1 (survived)
  - a **loss** earns no star.

### Speed & pause

Pause / 1× / 2×, per PRD 0001 §9. **Building is allowed while paused** (an unrestricted
accessibility assist). Speed is a playback multiplier over a fixed-rate sim, so a fixed input log
re-simulates identically at any speed.

### Art (light sprite set)

M1 uses a **small, original sprite set** — not commissioned art, and license-clean per ADR 0002:

- One **creep** sprite (a distinct silhouette), one **tower** sprite (reads as a 2 × 2 footprint),
  **board tiles** (floor / border-wall / entrance / exit markers), and a **cosmetic projectile**.
- **Placement & range feedback:** a shaded **range circle** on the hovered/selected tower; a
  **valid / invalid** placement tint (green / red) on the candidate footprint.
- **Accessibility (ADR 0003):** colorblind-safe palette and **dual-encoding** — state is never
  carried by color alone. Placement validity pairs its tint with a shape/icon cue; the range circle
  is a shape, not a hue; contrast meets the gate. (With only one creep and one tower, unit
  *differentiation* is trivial at M1; the discipline is set here for M2's roster.)

### Accessibility & i18n

- The **full ADR 0003 gate** applies from M1 (dual-encoding, contrast, keyboard/focus operation,
  the "strategic, not twitch" stance backed by pause + build-while-paused + speed).
- **All strings externalized** per ADR 0004 from the first UI.
- The **selectable-difficulty §3 waiver** (M1 ships Medium-only) is recorded in the accessibility
  checklist and each player-facing PR, per PRD 0001 §10.

## Stories (the build plan)

Seven stories, each ≈ one PR with its own done-criteria. Sim stories 1–5 land **before any pixels**,
so the deterministic core is provable on its own (world-hash stable, unit-tested); the client and
accessibility stories make it a public alpha.

1. **Board & grid (sim).** Cell grid + cell classes, the single-path M1 board expressed in the
   ADR 0007 ruleset, entrance/exit, and the pathfinding tie-break.
   *Done:* the sim loads the M1 board; entrance→exit reachability and a deterministic shortest path
   are computed; unit-tested; world-hash stable.
2. **Creep movement (sim).** Spawn from the wave schedule, 8-connected no-corner-cut travel toward
   the exit, leak at the exit costing a life.
   *Done:* a creep walks entrance→exit deterministically; a leak decrements lives; world-hash stable
   across a fixed input log.
3. **Placement, maze invariant & re-path (sim).** 2 × 2 build consuming bounty, buildable-cell and
   place-adjacent-not-on-creep checks, maze-invariant rejection, sell + haircut refund, dynamic
   re-path on every maze change.
   *Done:* build/sell works; invalid placements are rejected before applying; live creeps re-route
   the tick the maze changes; determinism intact.
4. **Combat (sim).** Single-target scheduled-impact, range acquisition, sticky "first" targeting,
   fire-time snapshot, kill → per-kill bounty.
   *Done:* a tower kills creeps in range; bounty is credited; scheduled impacts resolve in a
   deterministic total order; a wasted shot (target died/leaked) is handled.
5. **Wave lifecycle, win/loss & score (sim).** Countdown + call-early, spawn spacing, win/loss
   resolution, numeric score + star grade.
   *Done:* a full one-wave run resolves to a win or a loss with a **server-re-derivable** score and
   star grade.
6. **Render, input & HUD (client / Phaser).** The light sprite set, range circle, valid/invalid
   placement tint, build/sell input, HUD (bounty / lives / countdown / score), and pause
   (build-while-paused) / 1× / 2× controls.
   *Done:* the slice is playable end to end in the browser within the ADR 0005 performance budget.
7. **Accessibility gate + i18n (M1 conformance).** ADR 0003 checklist (dual-encoding, contrast,
   keyboard/focus, no-twitch) and ADR 0004 string externalization; record the selectable-difficulty
   §3 waiver.
   *Done:* the accessibility checklist passes and every player-facing string is externalized.

## Determinism & data

- Stories 1–5 are **pure `packages/sim` / `packages/engine`** surface — a pure function of
  `(seed, ruleset, boardId, tick inputs)`, no I/O / floats / `Math.random` / wall-clock (PRD 0001,
  ADR 0001/0006). Integer fixed-point throughout (256 units/tile).
- Every number above is **ruleset data** (ADR 0007): tuning bumps `rulesetHash`, not `simVersion`.
  The *shapes* are 0001's; M1 changes none of them, so `simVersion` is unaffected by tuning.
- Scoring is computed sim-side so the Phase-3 ladder can re-derive it (ADR 0006). Score, replay,
  and world-hash are identical at any playback speed for a fixed input log.
- M1 uses the `boardId` content identity. (The queued `level`→`board` vocab sweep — `levelId`→
  `boardId`, ADR 0007 title, schema — lands as its own focused change around this milestone so the
  gameplay code is born on the correct vocabulary.)

## Open questions / first-pass flags

Edges for reviewers — the values above are all tunable, but these are the deliberate judgment calls:

- **Starting lives = 10, not 20.** 20 was the other candidate; against a single 10-creep wave it is
  unloseable by leaking, which would leave the loss path untested in real play. 10 makes "leak the
  whole wave → loss" reachable (an undefended board loses). Revisit after the first playtest — a lower
  value would make *partial* leaks loseable, a harder but more instructive slice.
- **Spawn spacing (≈ 1 s) is a first-pass pick**; confirm it feels right once the wave is playable.
- **Star thresholds** are a fresh layer with no prior tuning; tune against real runs.
- **Balance as a whole** — M1 is deliberately winnable; genuine challenge tuning begins at M2–M3
  with more creeps and waves. M1 proves the loop, not the difficulty.
